Introduction

This section covers the basics of CSS layouts and responsive design.

Normal Document Flow

Normal document flow is the default way elements are arranged in a web page. Elements are stacked vertically, and each element takes up the full width of its parent container.

  • Block elements start on a new line and take up the full width available.
  • Inline elements do not start on a new line and only take up as much width as necessary.
  • Elements can be positioned using CSS properties like margin, padding, and border.
  • Understanding normal document flow is essential for creating layouts that are easy to read and navigate.

Display Properties

Explore different display properties like block, inline, and flex.

  • Block: Elements take up the full width available.
  • Inline: Elements only take up as much width as necessary.
  • Flex: Elements can be arranged in a flexible layout.
  • Grid: Elements can be arranged in a grid layout.
  • None: Elements are not displayed.
  • Inline-block: Elements are inline but can have width and height.
  • Float: Elements can be floated to the left or right.

Responsive Web Design

Responsive web design is about creating web pages that look good on all devices. It uses fluid grids and flexible images.

  • Fluid layouts: Use percentages for widths to create fluid layouts that adapt to different screen sizes.
  • Media queries: Apply different styles based on the device's characteristics, such as width, height, and orientation.
  • Flexible images: Use CSS to ensure images scale correctly within their containers.
  • Viewport meta tag: Use this tag to control the layout on mobile browsers.
  • Mobile-first design: Start designing for the smallest screens first, then add styles for larger screens.
  • Testing: Always test your designs on multiple devices and screen sizes to ensure a good user experience.
